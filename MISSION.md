# Mission: Production database operations

## Why

Learn to safely prepare, populate, and verify a production database for deployed web applications, starting with KnicksIQ. The outcome is being able to operate a release without guessing, exposing credentials, or leaving a public app pointed at incomplete data.

## Success looks like

- Explain the difference between a database schema migration and a data release.
- Run a migration, load a verified release artifact, and confirm readiness for a deployed application.
- Diagnose the difference between a live service and a ready service.

## Constraints

- Learn from short, practical lessons tied to real deployment work.
- Use the local Terminal to operate production databases; keep connection strings private.

## Out of scope

- Database design, query optimization, and building a migration framework from scratch.

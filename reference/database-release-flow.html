Database release flow

  1. Connect: set DB_URL only in your shell or deployment secret store.
  2. Migrate: apply versioned schema changes with alembic upgrade head.
  3. Validate: verify the release file’s hash and internal rules.
  4. Load: insert the release data transactionally.
  5. Activate: make one validated release the public application’s source of truth.
  6. Verify: check readiness, then exercise a real public query.

Mnemonic: structure first, data second, traffic last.
